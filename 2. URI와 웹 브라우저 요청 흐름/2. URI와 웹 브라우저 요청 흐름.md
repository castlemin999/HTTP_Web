# URI와 웹 브라우저 요청 흐름

## URI (Uniform Resource Identifier)

- 리소스를 식별하는 통합된 방법
- 로케이터(Locator)와 이름(Name) 또는 둘다 추가로 분류될 수 있다.

## URL (Uniform Resource Locator)

- 리소스가 있는 위치를 지정

### 문법

URL = 프로토콜 + 호스트명 + 포트 번호 + 패스 + 쿼리 파라미터

ex) https(프로토콜)://www.youtube.com(호스트명)/watch(패스)?v=iLqGzEkusIw(쿼리 파라미터)

## URN (Uniform Resource Name)

- 리소스에 이름을 부여
→ 이름만으로 실제 리소스를 찾을 수 있는 방법이 보편화 되지 않아 거의 사용되지 않음

## 웹 브라우저 요청 흐름

1. https://www.youtube.com/watch?v=iLqGzEkusIw
→ DNS 조회 (실제 IP 주소 ex. 211.231.284.92)
2. 웹브라우저에서 HTTP 요청 메시지 생성
3. TCP/IP 패킷 생성, HTTP 메시지 포함
4. 요청 패킷 서버에 전달, 서버에 도착
5. 서버에서 HTTP 응답 메시지 작성
6. 서버에서 웹브라우저로 응답 패킷 전달 (TCP/IP + 데이터(html, json, xml…))
7. 웹브라우저에 도달 후 데이터 활용(HTML 렌더링)
